## Step 2 — Define Location Datasets

Four location datasets are defined in the notebook:

| Dataset | Region | Purpose |
|---|---|---|
| `kerla_locations` | Kerala (Malappuram) | Flood prediction |
| `dhubri_locations` | Assam (Dhubri) | Flood prediction |
| `bihar_locations` | Bihar (multiple cities) | Heatwave prediction |
| `hyderabad_locations` | Telangana (Hyderabad) | Heatwave prediction |

Relief camps are auto-generated by shifting each flood location 1 km north and east to avoid overlap.

---

## Step 3 — Fetch Weather Data

```python
flood_locations  = kerla_locations    # Change to dhubri_locations if needed

df_flood = process_locations(flood_locations, mode="flood")

```

Weather is fetched from Open-Meteo API in real time.

---

## Step 4 — ML Classification

Labels are created dynamically based on a threshold:

**Flood**: score = `humidity × 0.5 + soil_moisture × 100`
- Above mean → Danger
- Below mean → Safe

**Heatwave**: based on heat index mean
- Above mean → Heatwave
- Below mean → Safe

Both Random Forest and Logistic Regression are trained. Random Forest is used as the primary model.

---

## Step 5 — Severity Scoring

Each danger zone receives a severity score:

**Flood Severity:**
| Parameter | Thresholds | Max Score |
|---|---|---|
| Wind Speed | 7 / 10 / 20 km/h | 3 |
| Precipitation | 3 / 5 / 7 mm | 3 |
| Humidity | 84% / 90% / 95% | 3 |
| Soil Moisture | 0.35 / 0.5 / 0.6 m³/m³ | 3 |

Total score → Low (≤2) / Medium (≤5) / High (>5)
